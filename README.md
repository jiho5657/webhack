# Hollowed

## 문제 설명

어느 암호학 연구원이 RSA 키를 생성하다가 실수로 한 소수의 상위 비트 일부를
로그에 남겨버렸다. 로그 파일 `challenge/output.txt` 를 입수했으니, 이 정보를
이용해 암호문을 복호화하고 플래그를 획득하자.

`challenge/output.txt` 에 들어 있는 공개 값은 다음과 같다.

- `N` : 2048비트 RSA 모듈러스
- `e` : 공개 지수 (`65537`)
- `c` : 플래그를 평문으로 한 암호문 (`c = flag^e mod N`)
- `p_high` : 모듈러스의 한 소인수 `p` 의 상위 `leak_bits` 비트
- `prime_bits`, `unknown_bits`, `leak_bits` : 비트 길이 파라미터

플래그 형식은 `MJSEC{...}` 이다.

## 힌트

1. `N` 은 두 개의 소수가 아니라 더 많은 소수의 곱이다. 각 소인수의 크기가
   전체 `N` 에서 차지하는 비율을 다시 계산해 보자.
2. Coppersmith 정리에서 쓰이는 `beta` 는 정해진 값이 아니라
   `log p / log N` 으로 결정된다. 교과서에서 자주 보는 `beta = 1/2` 설정을
   그대로 가져다 쓰면 이 문제는 풀리지 않는다.
3. SageMath 의 `small_roots` 기본 `epsilon` 값은 이 문제에서 여유가 부족해
   탐색이 실패한다. 직접 더 작은 값을 지정해야 하며, 대신 격자 차원이
   커지는 대가를 감수해야 한다.
